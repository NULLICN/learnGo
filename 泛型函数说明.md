# Go 泛型函数 - printLenCapValue 使用指南

## 📌 函数定义

```go
// printLenCapValue 打印任意类型切片的长度、容量和值（使用泛型）
func printLenCapValue[T any](slice []T) {
	fmt.Printf("长度为%d 容量为%d 值为%v\n", len(slice), cap(slice), slice)
}
```

## 🎯 核心特性

### 1. **泛型语法** `[T any]`
- `T` 是类型参数，代表任意类型
- `any` 是类型约束，表示接受任何类型
- 当调用函数时，Go 会自动推断 `T` 的具体类型

### 2. **支持的数据类型**
该函数可以接收任何类型的切片，包括：

| 数据类型 | 示例代码 | 输出 |
|---------|--------|------|
| `[]int` | `printLenCapValue([]int{1, 2, 3})` | 长度为3 容量为3 值为[1 2 3] |
| `[]string` | `printLenCapValue([]string{"a", "b"})` | 长度为2 容量为2 值为[a b] |
| `[]float64` | `printLenCapValue([]float64{3.14, 2.71})` | 长度为2 容量为2 值为[3.14 2.71] |
| `[]bool` | `printLenCapValue([]bool{true, false})` | 长度为2 容量为2 值为[true false] |
| `[]byte` | `printLenCapValue([]byte{'A', 'B'})` | 长度为2 容量为2 值为[65 66] |
| `[]struct` | `printLenCapValue([]User{...})` | 长度为... 容量为... 值为[...] |
| 任何类型 | `printLenCapValue([]T{...})` | ✓ 都支持 |

## 🔧 Go 泛型基础知识

### 什么是泛型？
泛型允许函数、类型和方法在定义时使用类型参数，而不是具体的类型。这样可以编写更通用、更灵活的代码。

### 泛型的优势
1. **代码复用** - 一个函数处理多种类型
2. **类型安全** - 编译时检查，不使用 `interface{}`
3. **性能** - 无需类型断言和反射
4. **可读性** - 代码意图清晰

### Go 1.18+ 泛型语法

#### 泛型函数
```go
func FunctionName[T any](param []T) {
    // 函数体
}
```

#### 泛型结构体
```go
type Stack[T any] struct {
    items []T
}
```

#### 泛型接口
```go
type Reader[T any] interface {
    Read() T
}
```

## 📊 类型参数约束

### `any` 约束
接受任何类型（等价于 `interface{}`）：
```go
func Print[T any](v T) {
    fmt.Println(v)
}
```

### 具体接口约束
只接受实现特定接口的类型：
```go
func Sum[T constraints.Integer](values []T) T {
    // ...
}
```

### 多个类型参数
```go
func Pair[K any, V any](key K, value V) {
    // ...
}
```

## ⚡ 性能对比

### 方案 1：使用 `interface{}`（不推荐）
```go
func printLenCapValue(slice interface{}) {
    v := reflect.ValueOf(slice)
    fmt.Printf("长度为%d 容量为%d 值为%v\n", v.Len(), v.Cap(), v.Interface())
}
```
**缺点**：
- 需要反射，性能较低
- 类型信息在运行时才检查
- 代码复杂度高

### 方案 2：使用泛型（推荐）✅
```go
func printLenCapValue[T any](slice []T) {
    fmt.Printf("长度为%d 容量为%d 值为%v\n", len(slice), cap(slice), slice)
}
```
**优点**：
- 编译时生成特定类型的代码
- 类型检查在编译期进行
- 性能与具体类型函数相同
- 代码简洁明了
